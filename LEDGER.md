- **Bugs found and fixed during implementation, not planning:** (1) *Chunking off-by-stride bug* â€” the initial loop always advanced by a fixed stride regardless of where the current window ended, so when the stride didn't evenly divide the text length it emitted a tiny, almost-entirely-duplicate trailing chunk; fixed by stopping once a window reaches the end of the text (caught by `test_produces_overlapping_windows`, a new reference case worth watching for in future windowing/pagination code). (2) *Test-only cross-user cookie leakage* â€” a test simulating two logged-in users on one `TestClient` failed because `resolve_token_from_request` checks the `session_token` cookie before the `Authorization` header, and `TestClient` shares one cookie jar across logins, so the most recently logged-in user's cookie silently overrode explicit Bearer headers set for an earlier-logged-in user. Not a production bug â€” real clients don't send conflicting cookies and headers for different identities â€” but a real pitfall for any future test simulating multiple concurrent identities on one `TestClient`; fixed by clearing the client's cookies right after capturing each login's token.
