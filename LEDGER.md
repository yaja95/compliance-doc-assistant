(2) *Test-only cross-user cookie leakage* — a test simulating two logged-in users on one `TestClient` failed because `resolve_token_from_request` checks the `session_token` cookie before the `Authorization` header, and `TestClient` shares one cookie jar across logins, so the most recently logged-in user's cookie silently overrode explicit Bearer headers set for an earlier-logged-in user. Not a production bug — real clients don't send conflicting cookies and headers for different identities — but a real pitfall for any future test simulating multiple concurrent identities on one `TestClient`; fixed by clearing the client's cookies right after capturing each login's token.
